🔧 技术架构

采用现代化的 Manifest V3 架构设计

Manifest V3

Chrome 扩展最新标准

Service Worker

后台服务脚本

Content Script

页面内容脚本注入

EventBus

事件驱动通信

Chrome Storage

数据持久化与同步

declarativeNetRequest

网络请求过滤修改

模块化架构设计

  • 功能模块独立 - reader-mode、video-mode、translator、search 相互独立
  • EventBus 通信 - 模块间通过事件总线通信，避免直接耦合
  • 中央主题管理 - 统一的主题管理系统，变化时广播至全部模块
  • 存储层抽象 - 统一的 storage 接口，自动处理本地/同步存储切换
  • 易于扩展 - 模块化设计使添加新功能变得简单快速

📚 核心模块说明

了解各个模块的职责和功能

🔙 Service Worker

职责： 监听扩展生命周期事件、管理 declarativeNetRequest 规则、定期清除浏览器 Service Workers、处理后台计时任务。

🔗 链接预览模块

职责： 监听页面链接交互、创建和管理预览浮窗、处理内容 iframe 加载、集成 Service Worker 清除机制。

📊 预览管理器

职责： 管理所有活跃预览窗口、处理窗口位置和大小、实现 Pin 功能、统计活跃/固定窗口数量。

🎨 UI 管理器

职责： 处理用户交互事件、管理浮窗上的按钮和控件、实现拖拽和缩放交互、消息传递给功能模块。

📖 阅读模式

职责： 提取网页主要内容、清除广告和干扰元素、优化排版和可读性、保存阅读状态。

🎬 视频模式

职责： 检测视频资源、优化视频播放体验、处理全屏显示、自适应分辨率。

🌐 翻译模块

职责： 快速翻译选中文本、支持多语言、浮窗显示翻译结果、缓存翻译记录。

🔍 搜索模块

职责： 快速搜索选中文本、支持多搜索引擎、自定义搜索配置、记录搜索历史。

🎨 主题管理器

职责： 加载和切换主题、应用主题样式、监听系统主题变化、广播主题变化事件。

❓ 常见问题

快速找到答案

Q: 如何预览链接？

有三种方式预览链接：

  • Ctrl+点击 - 按住 Ctrl 键点击链接
  • 右键菜单 - 右键链接选择"在预览窗口中打开"
  • 拖拽 - 拖拽链接到页面空白处
Q: 预览窗口不显示怎么办？
  1. 打开 chrome://extensions/
  2. 找到 NowView 扩展，点击刷新
  3. 打开浏览器控制台（F12）检查错误信息
  4. 如果仍未解决，清除浏览器缓存并重新加载
Q: 支持哪些网站？

NowView 支持大多数网站，包括但不限于：

  • GitHub、GitLab 等代码托管平台
  • 知乎、掘金等内容社区
  • YouTube、Bilibili 等视频平台
  • WordPress、Medium 等博客平台
  • 大多数新闻和信息网站

某些网站可能有特殊限制，我们持续优化支持。

Q: 如何切换主题？

点击 Chrome 右上角的 NowView 图标，在弹出窗口中选择⚙️，然后在设置页面选择你喜欢的主题（亮色/暗色/自动）。

Q: 我的数据会被上传吗？

不会。NowView 是完全本地化的扩展，所有数据都存储在你的浏览器中。我们不收集、存储或上传任何用户数据。

Q: 如何为项目贡献代码？
  1. Fork 项目到你的 GitHub 账户
  2. 创建新分支进行开发
  3. 提交 Pull Request
  4. 等待审查和合并

请查看 CONTRIBUTING.md 了解详细贡献指南。
